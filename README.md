# 🧠 TrendPulse - 맞춤형 기술/업무 트렌드 큐레이터

사용자가 설정한 직무(기구 설계, 자동화, IT, 전자, AI, 바이오 등)에 맞춰 최신 기술 동향, 해외 논문, 특허 소식, 교육 정보를 AI가 매일 아침 요약 브리핑해 주는 싱글페이지 웹 애플리케이션(SPA)입니다.

---

## 📅 개발 진행 내용

본 프로젝트는 프론트엔드 전용(HTML5, CSS3, Vanilla JS) 프로토타입으로 완성도 높은 비주얼과 인터랙션을 중심으로 구현되었습니다.

### 1. 🎨 프리엄 디자인 시스템 적용
- **글래스모피즘 UI**: 반투명 배경 블러 효과(`backdrop-filter: blur()`)를 활용하여 트렌디하고 세련된 카드 레이아웃 구현
- **다크/라이트 테마**: 딥 네이비(#0a0e27)의 딥 스페이스 배경 다크 모드와 심플하고 깔끔한 라이트 모드 간의 실시간 테마 스위칭 및 색상 변수 정의
- **마이크로 애니메이션**: 카드 호버 글로우 효과, 탭 전환 페이드인, 온보딩 단계 전환, 사이드바 폴딩 애니메이션 적용
- **배경 파티클 효과**: Canvas API를 활용하여 화면 전체에 부드럽게 흐르는 동적 파티클 효과 적용 (테마 색상과 연동)

### 2. 📡 실시간 대시보드 및 브리핑 기능
- **모닝 브리핑**: 접속 시간대에 따른 맞춤형 AI 환영 인사 및 실시간 요일/시계 구현
- **지표 카드 및 카운트업**: 오늘 제공되는 트렌드, 논문, 특허, 교육 수치에 3차 이징(Easing) 함수가 적용된 숫자 카운트업 애니메이션 탑재
- **데이터 시각화 (Pure Canvas API)**:
  - **기술 분야 레이더 차트**: 선택한 직무들의 트렌드 점수를 비교 분석하는 다각형 차트
  - **주간 트렌드 바 차트**: 요일별 트렌드 변화량을 그라디언트 및 둥근 모서리로 표현한 바 차트

### 3. 🔍 직무별 커스텀 콘텐츠 뷰
- **기술 트렌드 피드**: 관련도 점수(Critical/High/Medium/Low)에 따른 컬러 배지 적용 및 카드 클릭 시 펼쳐지는 아코디언식 AI 상세 요약 정보 탑재
- **논문 브리핑**: 최신 학술 논문의 저자, 저널, 인용 횟수 및 관련도 표시
- **특허 소식**: 타임라인 방식으로 정렬된 특허 정보와 출원인, 특허 번호 렌더링
- **교육 추천**: 플랫폼 정보, 난이도(초/중/고), 수강 소요 시간 및 추천 이동 링크 렌더링

### 4. 🔖 편리한 편의 기능
- **3단계 온보딩**: 첫 방문 시 사용자 유도 모달(소개 -> 직무 선택 -> 키워드 추천 및 추가)을 통해 초기 환경 자동 설정
- **북마크 시스템**: 마음에 드는 트렌드/논문/특허/교육 정보를 저장해 두고 모아볼 수 있는 영속성 보관 기능
- **환경 설정**: 사용자 이름 변경, 관심 직무 및 키워드 동적 추가/제거, 테마 스위치 제공

## 🛠️ 에러 조치 내용 (Troubleshooting Log)

개발 및 검증 과정에서 발생한 주요 오류와 불편 사항을 아래와 같이 개선 및 조치했습니다.

### 1. 북마크 탭 전환 시 `ReferenceError` 발생 해결
- **문제 현상**: 북마크 섹션 내의 카테고리 탭(전체/트렌드/논문 등)을 클릭하여 전환할 때, 브라우저 환경에 따라 `ReferenceError: event is not defined`가 발생하며 UI가 멈추는 현상 확인.
- **원인 분석**: `renderBookmarks` 함수 내부에서 전역 `event` 객체를 부적절하게 직접 참조하여 탭의 active 클래스를 다루고 있었음.
- **조치 내용**: 
  - 각 탭 버튼 엘리먼트에 `data-type` 어트리뷰트 추가.
  - JS 함수를 이벤트 객체 의존 없이 매개변수로 넘어온 `type`과 `dataset.type`을 비교하여 `classList.toggle`로 탭 활성화 상태를 안전하게 제어하도록 리팩토링 완료.

### 2. 설정 화면 변경 내용 유실 방지 (실시간 자동 저장)
- **문제 현상**: 프로필 설정 화면에서 관심 직무 및 키워드를 토글/수정한 후 하단의 "저장" 버튼을 명시적으로 누르지 않고 탭을 이동하거나 페이지를 새로고침하면 변경 사항이 유실되는 불편함 발생.
- **원인 분석**: 상태 변경 즉시 `localStorage`에 데이터가 쓰이지 않고 별도 저장 이벤트 핸들러(`saveJobSettings`, `saveKeywordSettings`)를 거치게 설계되어 있었음.
- **조치 내용**: 
  - 직무를 선택/해제하거나 키워드를 추가/제거(`toggleSettingsCategory`, `handleTagInput`, `removeTag`, `addSuggestedTag`)할 때마다 마지막 단계에서 `saveSettings()`를 동기 호출하도록 구조 변경.
  - UI 상에서 불필요하고 혼란을 주는 개별 "저장" 버튼을 제거하고, 자동 저장 안내 메시지 추가로 UX 대폭 향상.

### 3. 3개 미만 직무 선택 시 대시보드 레이더 차트 오류 대응
- **문제 현상**: 온보딩 또는 설정에서 관심 직무를 1개 또는 2개만 선택했을 때, 대시보드의 레이더 차트가 전혀 그려지지 않아 빈 캔버스가 흉하게 노출되는 문제.
- **원인 분석**: 다각형을 그리는 레이더 차트의 특성상 축이 최소 3개(3차원 이상) 확보되어야 연산이 가능하여 `n < 3`인 경우 그냥 리턴해버렸기 때문.
- **조치 내용**:
  - `drawRadarChart` 함수 내에 예외 조건 분기 추가.
  - 선택된 직무가 3개 미만인 경우, 빈 캔버스 중앙에 테마 컬러와 어울리는 다국어 서체의 가이드 문구(**"직무를 3개 이상 선택하셔야 분야별 레이더 차트가 표시됩니다."**)를 동적으로 렌더링하도록 조치하여 UI 무너짐 방지.

### 4. Git 커밋 생성 시 `Author identity unknown` 오류 대응
- **문제 현상**: 로컬 저장소 생성 후 첫 커밋을 생성할 때, Git 환경의 글로벌 `user.email` 및 `user.name` 정보가 등록되지 않아 커밋 실행이 실패함.
- **조치 내용**: 
  - 임시 로컬 구성 명령어(`git config user.email "..."`, `git config user.name "..."`)를 실행하여 해당 저장소 범위에서 인증을 우회하고 정상적으로 첫 커밋을 성공시킴.
